System Initialization Functions#back to top

UTCInitialize #back to top

                                    utc_result UTCAPI UTCInitialize (const UTC_INIT* pInitData);

This function is used to initialize the UltraSee’s Ultrasound Imaging System. It should always be called before any other functions are called.

@input: a UTC_INIT pointer pInitData

@return: utc_result indicating if the system has successfully initialized.


The pInitData is structured as follows:

                                    typedef struct tagUTC_INIT
                                    {
	                                    HWND hWnd;
	                                    UINT nUserMsg;
	                                    UTC_EVENT_CALLBACK eventCallback; 
                                    } UTC_INIT;

Where hWnd is the Window handle to receive probe message, nUserMsg is the message code to be used when probe message is posted to hWnd, and eventCallback is the probe event callback function. The callback function is used in none-windows platforms, and if it is not used, then it should be set to NULL.

UTCAttachProbe #back to top

                                    UTCAPI utc_result UTCAttachProbe(UTC_PROBE_HANDLER* pProbeHandler, const TCHAR* pSettingName = NULL, bool bUseVirtualProbe = false);

When a probe is attached to the computer, the application will receive a USB attached event from OS. This function is used to attach to a probe in SDK using input configurations.

@input: a UTC_PROBE_HANDLER pointer pProbeHandler, a constant string pSettingName (by default set to NULL), a Boolean value bUseVirtualProbe (by default set to false)

pProbeHandler is a callback handler to receive images.

pSettingName is the name of probe settings the user wants to attach to the probe. If it is NULL, then the default settings will be used.

bUseVirtualProbe defines if a real probe is attached (false) or a virtual probe is used for simulation (true).

@return: utc_result indicating if it has successfully attached to a probe in SDK.

The pProbeHandler is structured as follows:

                                    typedef struct tagUTC_PROBE_HANDLER
                                    {
	                                    UTC_FRAME_CALLBACK FrameCallback;
	                                    void*	pUserData;
                                    } UTC_PROBE_HANDLER;

The FrameCallback is a pointer that points to the callback handler function, and the handle function will take pUserData as input.

UTCAdjustTGC #back to top

                                    UTCAPI utc_result UTCAdjustTGC(int nTGCIndex, int nTGCValue);

This function is used to adjust probe TGC.

@input: two integer values, nTGCIndex and nTGCValue>

nTGCIndex is the index of TGC position. It should be in range 0 – N (depends on how many TGC segments supported).

nTGCValue is the TGC value and should be in range 0 – 100 (100% gain).

@return: utc_result indicating if it has successfully adjusted probe TGC.


UTCGetTGC #back to top

                                    UTCAPI utc_result UTCGetTGC(int* pTGCValues, int& nSize);

This function is used to get all TGC settings.

@input: an integer pointer pTGCValues and an integer reference nSize

pTGCValues is a buffer to receive TGC values.

nSize is the maximum size of TGC values buffer. It will return actual TGC value size fetched, for example, if it is a 4 segments TGC, then 4 values are returned. If it is a 2 segments TGC, then 2 values are returned.

@return: utc_result indicating if it has successfully fetched the probe TGC values.

UTCEnableRFRecording #back to top

                                    UTCAPI utc_result UTCEnableRFRecording(bool bEnable);

This function is used to enable/disable RF data recording.

If the recording is turned on, RF data are recorded in every freeze. Last N seconds frame data are recorded into a new file in folder "record" or configured location.

@input: a Boolean value bEnable

if bEnable is true, then RF data recording will be enabled. else if bEnable is false, then RF data recording will be disabled.

@return: utc_result indicating if it has successfully enabled/disabled RF data recording.
